# 幼儿园排课系统 - 产品文档

> 本文档记录幼儿园 Web 端排课系统的产品定义、页面结构及交互逻辑。本文档仅涉及纯前端页面功能，不涉及后端逻辑与数据持久化。

---

# 排课模块

## 一、产品概述

### 1.1 产品定位
「幼儿园排课系统」是一款面向幼儿园教师的 Web 端可视化排课工具，通过拖拽方式快速将活动安排到每日课表中，帮助教师直观地完成周课程计划的制定与调整。

### 1.2 目标用户
- 幼儿园班主任 / 任课教师
- 教学组长 / 教务管理人员

### 1.3 核心功能
- **活动列表**：分「我的」「园所」「平台」三类来源展示，以卡片形式呈现，支持拖拽。
- **双重筛选**：来源筛选（我的/园所/平台/全部）+ 领域筛选（健康/语言/社会/科学/艺术/全部）。
- **文本搜索**：支持按活动名称关键词搜索。
- **课表视图**：以大表格形式展示一周五个工作日的活动安排，横轴为日期，纵轴为上课时段。
- **日期导航**：支持周级切换（上一周/下一周）、一键回到本周、输入指定日期精准定位。
- **主题提示**：同一自然周的五个工作日使用统一颜色标识，鼠标悬浮日期头部可查看本月主题、本周主题与今日主题。
- **活动详情**：点击活动卡片详情按钮，查看活动完整信息。
- **负责人配置**：活动排入课表后，在单元格内卡片下方配置负责人，更换活动时不更改负责人。

### 1.4 使用场景
- 学期初批量制定周课程计划
- 临时调课或替换某节活动
- 查看本周 / 下周活动安排

### 1.5 范围说明
本文档仅描述前端页面功能与交互规则，不涉及后端接口设计、数据库存储、权限控制及打印导出等功能。

## 三、活动列表

### 3.1 活动来源分类

活动列表按来源分为三类，通过顶部第一行筛选标签进行切换显示。

| 分类 | 说明 | 示例 |
|------|------|------|
| **我的** | 当前教师个人创建或收藏的活动 | 班主任自己设计的特色课 |
| **园所** | 所在幼儿园统一配置的活动库 | 园区统一采购的标准化课程 |
| **平台** | 第三方平台接入的活动资源 | 教育局推荐课程、合作平台课程 |

**来源筛选标签交互**：
- 标签位于「活动列表」标题下方第一行，横向平铺排列：「全部」「我的」「园所」「平台」
- 「全部」为默认选中态，展示所有来源的活动卡片
- 点击某一标签后，仅展示该来源的活动卡片，其他来源隐藏
- 选中态：主色背景 + 白色文字；未选中态：浅色背景 + 常规文字

### 3.2 活动领域分类（五大领域）

在活动来源筛选标签下方，增加第二行领域筛选标签，按《3-6岁儿童学习与发展指南》五大领域划分。

| 领域 | 说明 |
|------|------|
| **健康** | 身心状况、动作发展、生活习惯与生活能力 |
| **语言** | 倾听与表达、阅读与书写准备 |
| **社会** | 人际交往、社会适应 |
| **科学** | 科学探究、数学认知 |
| **艺术** | 感受与欣赏、表现与创造 |

**领域筛选标签交互**：
- 标签位于来源筛选标签下方，横向平铺排列：「全部」「健康」「语言」「社会」「科学」「艺术」
- 「全部」为默认选中态
- 点击某一标签后，仅展示该领域的活动卡片
- 来源筛选与领域筛选为**交集关系**，同时生效

### 3.3 活动类型分类

在活动领域筛选标签下方，增加第三行活动类型筛选标签。

| 活动类型 | 说明 |
|----------|------|
| **生活** | 培养日常生活自理能力、卫生习惯等 |
| **运动** | 体育锻炼、体能训练、户外活动等 |
| **游戏** | 角色扮演、建构游戏、益智游戏等 |
| **学习** | 知识探索、认知发展、阅读学习等 |

**活动类型筛选标签交互**：
- 标签位于领域筛选标签下方，横向平铺排列：「全部」「生活」「运动」「游戏」「学习」
- 「全部」为默认选中态
- 点击某一标签后，仅展示该活动类型的活动卡片
- 来源筛选、领域筛选、活动类型筛选三者互为**交集关系**，同时生效

### 3.4 文本搜索

在两行筛选标签下方，放置文本搜索框。

| 元素 | 说明 |
|------|------|
| 搜索输入框 | 占位文字「搜索活动名称」，支持实时模糊匹配 |
| 清除按钮 | 输入框右侧 `✕` 图标，点击清空搜索内容并恢复完整列表 |
| 搜索范围 | 仅针对活动名称 `name` 字段进行匹配 |

**搜索交互**：
- 输入时实时过滤列表，展示匹配的活动卡片
- 无匹配结果时展示空状态提示「未找到相关活动」
- 搜索与筛选标签为**交集关系**，同时生效

### 3.5 活动卡片字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | string | 是 | 活动唯一标识 |
| `name` | string | 是 | 活动名称，如「创意美术」 |
| `type` | string | 是 | 活动领域，如「艺术」「健康」「语言」 |
| `category` | string | 否 | 活动类型，如「生活」「运动」「游戏」「学习」 |
| `source` | string | 是 | 活动来源：`my`（我的）/ `institution`（园所）/ `platform`（平台） |
| `duration` | number | 是 | 活动时长，单位：分钟，如 30 / 40 / 60，用于计算占用单元格数，**卡片上不展示** |
| `color` | string | 是 | 活动卡片主题色，如 `#FF9F43` |
| `teacher` | string | 否 | 默认授课教师姓名 |
| `icon` | string | 否 | 活动图标 URL 或图标类名 |
| `description` | string | 否 | 活动简介，用于详情弹窗 |
| `targetAge` | string | 否 | 适用年龄段，如「3-4岁」 |
 | `objectives` | string | 否 | 教学目标 |
| `materials` | array | 否 | 课程资料列表，每项包含 `name`（资料名称）与 `url`（在线查看地址），用于活动详情弹窗 |

### 3.5 活动卡片展示内容

活动卡片上**仅展示以下信息**（不展示活动时长、不展示教师）：

```
┌────────────────────────────┐
│  创意美术             [ⓘ]  │  ← 活动名称 + 右上角详情按钮
│  ━━━━━━━━━━━━━━━━━━━━━━━━  │  ← 顶部色条（活动主题色）
│  [艺术]                    │  ← 领域标签
└────────────────────────────┘
```

| 展示元素 | 说明 |
|----------|------|
| 顶部色条 | 活动主题色 `color`，高度 4px，圆角顶部 |
| 活动名称 | 13px 加粗，左对齐 |
| 详情按钮 | 右上角 `ⓘ` 图标按钮，hover 时显示，点击打开活动详情 |
| 领域标签 | 活动领域 `type`，小号标签样式，带浅色背景 |

> **说明**：活动卡片**不展示授课教师**。教师（负责人）信息在活动被排入课表后，通过课表单元格内的负责人下拉选择器进行配置。

### 3.6 活动状态

| 状态 | 说明 |
|------|------|
| 未安排 | 活动尚未被拖入课表，正常显示在活动列表中 |
| 已安排 | 活动已被拖入课表，活动卡片可减少透明度或置灰（可选） |

### 3.7 列表交互

- **筛选来源**：点击第一行来源筛选标签，切换展示全部/我的/园所/平台活动
- **筛选领域**：点击第二行领域筛选标签，切换展示全部/健康/语言/社会/科学/艺术活动
- **筛选活动类型**：点击第三行活动类型筛选标签，切换展示全部/生活/运动/游戏/学习活动
- **文本搜索**：在搜索框输入关键词，实时过滤活动名称
- **清除搜索**：点击搜索框右侧 `✕` 清除按钮，清空关键词并恢复完整列表
- **拖拽发起**：鼠标按住活动卡片并移动，卡片进入拖拽态，跟随鼠标移动
- **拖拽中**：原卡片位置保留占位符或降低透明度，提示用户源位置
- **拖拽释放**：在课表有效区域释放后执行放置逻辑；在非有效区域释放则卡片弹回列表
- **查看详情**：点击卡片右上角 `ⓘ` 详情图标按钮，弹出活动详情弹窗

### 3.8 活动详情弹窗

点击详情按钮后弹出模态弹窗，展示活动完整信息：

| 字段 | 展示方式 |
|------|----------|
| 活动名称 | 弹窗标题 |
| 活动领域 | 标签样式 |
| 活动类型 | 标签样式，如「生活 / 运动 / 游戏 / 学习」 |
| 活动来源 | 文本，如「我的 / 园所 / 平台」 |
| 活动简介 | 多行文本区域 |
| 教学目标 | 多行文本区域 |
| 课程资料 | 资料名称列表，每项为可点击链接，点击后在新标签页打开对应文件 |

**课程资料字段**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `materials` | array | 否 | 课程资料列表，每项包含 `name`（资料名称）与 `url`（在线查看地址） |

**课程资料交互**：
- 资料以链接形式平铺展示，如「📄 创意美术课件.pdf」
- 点击资料名称后通过 `window.open` 在新标签页打开 `url`
- 无资料时展示「暂无资料」

弹窗底部提供「关闭」按钮。

## 四、课表区域

### 4.1 坐标定义

- **横轴（X 轴）**：日期，仅展示周一到周五，共 5 列。
- **纵轴（Y 轴）**：上课时段，固定为每日四个时段：08:00-09:00、09:00-10:00、10:00-11:00、14:00-15:00。
- **单元格**：每个日期列与每个时段行的交叉位置，可放置 0 个或 1 个活动。

### 4.2 日期导航

| 操作 | 交互说明 |
|------|----------|
| 点击「上一周」 | 当前日期减去 7 天，重新计算所在周的周一到周五并渲染课表 |
| 点击「回到本周」 | 重置为系统当天日期，重新计算所在周的周一到周五并渲染课表 |
| 点击「下一周」 | 当前日期加上 7 天，重新计算所在周的周一到周五并渲染课表 |
| 日期输入框 | 输入格式 `YYYY-MM-DD`，校验通过后定位到该日期所在周的周一 |

按钮顺序固定为：**上一周｜回到本周｜下一周｜日期选择**

### 4.3 日期展示规则

- 课表仅显示工作日（周一到周五），不显示周六、周日。
- 日期格式为 `MM-DD 周X`，如「06-09 周二」。
- 同一自然周的五个工作日日期头部使用统一的背景色。
- 自然周的判定：以周一为每周第一天，按月份内第几周进行颜色映射。
- **今日高亮**：若当前周包含「今天」，则今天的日期头部额外添加深色边框或文字加粗标识。

### 4.4 时段展示规则

- 时段固定为四个：08:00-09:00、09:00-10:00、10:00-11:00、14:00-15:00。
- 时段格式为 `HH:mm-HH:mm`，如「08:00-09:00」「09:00-10:00」。
- 纵轴单元格直接展示完整时段区间，不再只展示起始时间。

### 4.5 课表单元格字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `date` | string | 是 | 日期，格式 `YYYY-MM-DD` |
| `weekday` | string | 是 | 星期几，如「周一」 |
| `timeSlot` | string | 是 | 时段，如 `09:00-10:00` |
| `activityId` | string | 否 | 已排活动 ID，空表示空闲 |
| `activityName` | string | 否 | 已排活动名称 |
| `activityColor` | string | 否 | 已排活动颜色 |
| `responsiblePerson` | string | 否 | 负责人，**通过单元格内卡片下方下拉选择器配置** |
| `isConflict` | boolean | 否 | 是否冲突，默认 false |

## 五、周主题 / 月主题

### 5.1 主题字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `month` | string | 是 | 所属月份，格式 `YYYY-MM` |
| `monthTheme` | string | 是 | 本月主题，如「夏天来了」 |
| `weekNumber` | number | 是 | 本月第几周，如 1 / 2 / 3 / 4 |
| `weekStartDate` | string | 是 | 本周开始日期（周一），`YYYY-MM-DD` |
| `weekEndDate` | string | 是 | 本周结束日期（周五），`YYYY-MM-DD` |
| `weekTheme` | string | 是 | 本周主题，如「认识小动物」 |
| `dailyThemes` | object | 否 | 每日主题映射，`{ "YYYY-MM-DD": "今日主题" }` |

### 5.2 悬浮提示

- **触发条件**：鼠标悬浮在日期表头（某一列的 `MM-DD 周X` 区域）时触发。
- **展示内容**：
  - 本月主题：{monthTheme}
  - 本周主题：{weekTheme}
  - 今日主题：{dailyTheme}（若 `dailyThemes` 中无该日期记录，则默认展示 `weekTheme`）
- **关闭条件**：鼠标移出日期表头区域时关闭。
- **展示形式**：白色圆角卡片，带小三角箭头指向触发日期头部。

悬浮窗内容示例：

```
本月主题：夏天来了
本周主题：认识小动物
今日主题：认识小猫咪
```

### 5.3 主题展示规则

- 同一自然周的 5 个工作日表头使用相同的背景色，颜色按月内第几周循环。
- 第 1 周：淡蓝色 `#E6F7FF`
- 第 2 周：淡绿色 `#F6FFED`
- 第 3 周：淡黄色 `#FFF7E6`
- 第 4 周：淡紫色 `#F9F0FF`
- 第 5 周及以后：循环复用或单独定义

## 六、核心交互

### 6.1 拖拽排课状态流转

```
[1] 拖拽开始（Drag Start）
    │
    ▼
    • 用户长按/点击并拖动左侧活动卡片
    • 活动卡片进入拖拽态：缩小、透明度降低、跟随鼠标
    • 记录拖拽源数据：activityId、activityName、duration、color

[2] 进入课表区域（Drag Enter）
    │
    ▼
    • 拖拽物进入课表单元格上方
    • 目标单元格高亮（边框变蓝/背景变浅蓝）
    • 根据活动 duration 计算占用单元格数量
    • 若跨多个单元格，则高亮连续目标区域

[3] 在单元格上移动（Drag Over）
    │
    ▼
    • 实时判断目标区域是否可放置
    • 若目标区域已有活动 → 高亮变红，提示冲突
    • 若目标区域空闲 → 保持蓝色高亮

[4] 放置（Drop）
    │
    ▼
    • 用户释放鼠标，触发放置
    • 执行冲突校验
        ├─ 通过：将活动数据写入目标单元格及后续连续单元格
        │         负责人字段为空，触发课表重绘
        │         课表单元格内卡片下方展示「负责人」占位态
        └─ 目标单元格已占用：弹出确认弹窗，询问「替换当前活动」或「取消」
            ├─ 点击「替换」：仅替换活动信息，**保留原有负责人信息**，触发课表重绘
            └─ 点击「取消」：关闭弹窗，活动卡片弹回活动列表

[5] 配置负责人
    │
    ▼
    • 活动被排入课表后，单元格内活动卡片下方展示负责人区域
    • 点击下拉选择器，展开负责人列表（数据来源：园所教师库）
    • 选中负责人后，下拉器更新展示所选负责人姓名
    • 负责人信息随课表数据一并保存
    • **替换活动时，负责人信息保持不变**
```

### 6.2 日期切换逻辑

```
[1] 点击「上一周」
    │
    ▼
    • currentDate 减去 7 天
    • 重新计算 weekStartDate / weekEndDate
    • 重新渲染日期头部与课表内容

[2] 点击「回到本周」
    │
    ▼
    • currentDate 重置为系统当天日期
    • 重新计算 weekStartDate / weekEndDate
    • 重新渲染日期头部与课表内容

[3] 点击「下一周」
    │
    ▼
    • currentDate 加上 7 天
    • 重新计算 weekStartDate / weekEndDate
    • 重新渲染日期头部与课表内容

[4] 日期输入框输入
    │
    ▼
    • 校验输入格式（YYYY-MM-DD）
    • 定位到该日期所在周的周一
    • 重新计算并渲染本周数据
```

### 6.3 冲突校验规则

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 目标单元格已占用 | 目标时段及后续连续时段中任意一格已有活动 | 弹出确认弹窗，提示「该时段已有活动，是否替换？」，提供「替换」和「取消」按钮 |
| 超出当天时间范围 | 活动 duration 跨越当天放学时间 | 「活动时长超出当日可排课时间」 |
| 跨天放置 | 活动不可跨日期放置 | 「活动不能跨天安排」 |
| 同一活动重复安排 | 同一天同一活动已存在（可选） | 「当天已安排该活动」 |

### 6.4 异常反馈

| 场景 | 反馈方式 |
|------|----------|
| 拖拽冲突（目标单元格已占用） | 目标区域变红 + 释放后弹出确认弹窗，询问「替换当前活动」或「取消」 |
| 日期输入非法 | 输入框边框变红 + 提示「请输入正确日期格式」 |
| 放置成功 | 活动卡片落入单元格，轻微缩放动画 |
| 放置失败 | 活动卡片弹回原位，显示失败原因 |

# 时段管理模块

> 本模块为幼儿园提供自定义时段管理能力，支持为不同年段、不同地区配置差异化的教学活动时段。配置好的时段集合将作为排课系统的「时段来源」供课表引用。

## 一、产品定位

「时段管理」是排课系统的子模块，面向教务管理人员或班主任，用于维护幼儿园各班级可用的上课时段方案。一套时段方案包含多个连续的时段区间，如「09:00-10:00」「10:00-11:00」等。

## 二、页面结构

### 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  时段管理                                                │
├─────────────────────────────────────────────────────────┤
│  [🔍 时段名称]        [全部 ▼]        [ + 新增时段 ]     │
├─────────────────────────────────────────────────────────┤
│  ┌──────────────┬─────────────────────┬──────────────┐  │
│  │ 时段名称     │ 描述                │ 操作         │  │
│  ├──────────────┼─────────────────────┼──────────────┤  │
│  │ 标准作息     │ 适用于小班的标...    │ [编辑][停用] │  │
│  │ [启用]       │                     │              │  │
│  ├──────────────┼─────────────────────┼──────────────┤  │
│  │ 午休专用时段 │ 仅用于午休及安...    │ [编辑][启用] │  │
│  │ [停用]       │                     │              │  │
│  ├──────────────┼─────────────────────┼──────────────┤  │
│  │ 大班扩展时段 │ 大班下午增加兴...    │ [编辑][停用] │  │
│  │ [启用]       │                     │              │  │
│  └──────────────┴─────────────────────┴──────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 区域说明

| 区域 | 位置 | 说明 |
|------|------|------|
| 顶部标题栏 | 页面最上方 | 模块标题「时段管理」 |
| 搜索操作区 | 标题栏下方 | 包含时段名称搜索框、状态下拉筛选、「新增时段」按钮 |
| 时段表格区 | 页面主体 | 以表格形式展示时段集合，每行展示名称、描述、操作 |

## 三、时段字段信息

### 时段集合（TimePeriodSet）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | string | 是 | 时段集合唯一标识 |
| `name` | string | 是 | 时段名称，如「标准作息」「午休专用时段」 |
| `description` | string | 否 | 时段描述，说明适用场景 |
| `status` | string | 是 | 状态：`active`（启用）/ `inactive`（停用），默认 `active` |
| `periods` | array | 是 | 时段列表，每项为一个时段区间 |

### 时段子项（PeriodItem）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `index` | number | 是 | 时段序号，从 1 开始递增 |
| `timeRange` | string | 是 | 时段区间，格式 `HH:mm-HH:mm`，如 `09:00-10:00` |

## 四、表格展示规则

### 表格列定义

| 列名 | 宽度 | 说明 |

| 操作 | 160px | 编辑按钮 + 停用/启用按钮 |

**时段名称列**
- 名称：14px 加粗，`#262626`
- 状态标签：名称右侧紧跟小标签，启用为蓝色 `#1890FF` 背景 10% 透明度 + 蓝色文字，停用为灰色 `#8C8C8C` 背景 + 灰色文字
- 标签圆角 4px，内边距 1px 6px，字号 12px

**固定字段**

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|

| 时段描述 | 文本输入 | ❌ 否 | 不能超过 100 字 |

**可重复添加字段：时段列表**

| 属性 | 说明 |
|------|------|
| 展示形式 | 纵向列表，每一项包含「时段{index}」标签、时间区间输入框、删除按钮 |
| 时间区间输入 | 格式 `HH:mm-HH:mm`，校验时间合法性（结束时间必须晚于开始时间） |
| 添加操作 | 表单底部「+ 添加时段」按钮，点击后追加一个新的空白时段项 |
| 删除操作 | 每项右侧 `✕` 按钮，点击后移除该时段项；删除后下方时段序号自动重排 |
| 最少数量 | 至少保留 1 个时段项；仅剩 1 项时，隐藏该时段项的删除按钮 |
| 最多数量 | 最多 10 个时段项，达到上限后「+ 添加时段」按钮置灰并提示「最多添加 10 个时段」 |

### 时段列表交互规则

1. **添加时段**：点击「+ 添加时段」，在列表末尾追加一项，默认 `timeRange` 为空，输入框获得焦点。
2. **删除时段**：点击某一项右侧 `✕`，该项从列表移除，后续项的 `index` 自动递减并重新展示（如删除时段2，原时段3变为时段2）。
3. **时间校验**：输入框失焦时校验格式与逻辑：
   - 格式不合法（非 `HH:mm-HH:mm`）→ 输入框边框变红，提示「请输入正确的时间格式，如 09:00-10:00」
   - 结束时间 ≤ 开始时间 → 边框变红，提示「结束时间必须晚于开始时间」
   - 时段重叠：与列表中其他时段区间存在重叠 → 边框变红，提示「时段区间不能重叠」

# 学期管理模块

> 本模块为幼儿园提供学期信息管理能力，支持维护各学年的学期起止时间，供排课系统、活动管理等其他模块引用。学期状态根据当前日期与起止时间自动判定。

## 一、产品定位

「学期管理」是排课系统的子模块，面向教务管理人员，用于维护幼儿园的学期时间线。每个学期包含名称、开始时间、结束时间，系统根据当前日期自动计算并展示学期状态（未开始 / 进行中 / 已结束）。

| 字段名 | 类型 | 必填 | 说明 |

> 本模块为幼儿园提供学期级的计划可视化能力，以画布+思维导图的形式展示学期内各自然月、自然周的主题与计划，支持层级浏览和编辑。

## 一、产品定位

「学期计划」是排课系统的子模块，面向教务管理人员和班主任，用于宏观规划和管理整个学期的教学节奏。页面以三级卡片结构（学期→月份→周）直观呈现计划体系，帮助教师快速把握教学主线。

## 四、学期选择规则

### 数据来源

学期下拉列表的数据来源于「学期管理」模块中创建的所有学期，包含学期 ID 与学期名称。

### 默认选中规则

1. **第一优先级**：遍历所有学期，根据当前日期计算状态，**优先选中状态为「进行中」的学期**。
2. **第二优先级**：若不存在「进行中」的学期，则选择 `startDate` **开始日期最新**的学期（即时间上距离当前最近的学期）。

### 切换规则

- 下拉选择其他学期后，画布区域立即重新渲染，展示对应学期的计划数据。
- 若所选学期暂无计划数据，画布区域展示空状态「暂无学期计划数据，请先选择学期」。

## 五、画布展示规则

### 三级卡片结构

画布内以**从上到下的层级结构**展示三级卡片：

| 层级 | 卡片样式 | 位置 | 内容 |
|------|----------|------|------|
| 学期级 | 最大，蓝色渐变背景，白色文字，圆角 12px | 画布顶部居中 | 学期名称 + 学期主题 + 编辑按钮 |
| 月份级 | 中等，白色背景，彩色边框，圆角 10px | 学期下方，横向排列 | 月份名称 + 主题 + 计划摘要 + 编辑按钮 |
| 周级 | 最小，浅绿色背景，圆角 8px | 月份下方，横向排列 | 第几周 + 主题 + 日期范围 |

### 连接线

- 学期卡片与月份卡片之间、月份卡片与周卡片之间，使用**垂直向下的虚线/实线 + 箭头**表示层级从属关系。
- 线条颜色：`#D9D9D9`，宽度 2px。

### 月份颜色循环

不同月份使用不同主题色边框进行视觉区分，按顺序循环使用：

| 序号 | 颜色 | 色值 |
|------|------|------|
| 0 | 蓝 | `#1890FF` |
| 1 | 绿 | `#52C41A` |
| 2 | 橙 | `#FA8C16` |
| 3 | 紫 | `#722ED1` |
| 4 | 粉 | `#EB2F96` |

### 卡片内容展示规则

**学期卡片**
- 标签：「学期主题」
- 标题：学期名称（18px 加粗）
- 主题：学期主题（13px，超出宽度省略号）
- 操作：编辑按钮

**月份卡片**
- 标签：月份名称（11px，与边框同色）
- 标题：月份主题（15px 加粗）
- 计划：计划内容前 14 字 + 省略号
- 操作：编辑按钮

**周卡片**
- 标签：第 X 周（11px，绿色）
- 标题：周主题（12px 加粗，最大宽度 72px，超出省略号）
- 日期：`MM-DD~MM-DD`

## 六、卡片交互

### 点击卡片（非编辑按钮区域）

点击任意层级的卡片主体（除编辑按钮外），弹出**详情弹窗**，展示该卡片的完整信息：

| 卡片类型 | 详情弹窗标题 | 展示字段 |
|----------|--------------|----------|
| 学期 | 学期详情 | 学期名称、学期主题、学期计划 |
| 月份 | 月份详情 | 所属学期、月份、主题名称、计划内容 |
| 周 | 周详情 | 所属月份、时间范围、主题名称、计划内容 |

空字段展示为「—」。

### 点击编辑按钮

点击卡片上的「编辑」按钮，弹出**编辑弹窗**，允许修改对应层级的主题和计划内容。

**可编辑字段**

| 卡片类型 | 可编辑字段 | 校验规则 |
|----------|------------|----------|
| 学期 | 学期主题、学期计划 | 无必填校验 |
| 月份 | 主题名称（必填）、计划内容 | 主题名称不能为空 |
| 周 | 主题名称（必填）、计划内容 | 主题名称不能为空 |

**只读字段**：学期名称、月份、时间范围等系统自动关联的字段为只读，灰色背景展示。

## 七、编辑弹窗

### 触发方式

- 点击学期/月份卡片上的「编辑」按钮 → 打开编辑弹窗

### 弹窗表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标识字段 | 文本输入（只读） | — | 根据类型展示学期名称/月份/时间范围，灰色背景不可编辑 |
| 主题名称 | 文本输入 | 月份/周必填 | 不能超过 50 字 |
| 计划内容 | 多行文本 | 否 | 不能超过 500 字 |

### 弹窗底部操作

| 按钮 | 说明 |
|------|------|
| 取消 | 关闭弹窗，不保存任何变更 |
| 确认保存 | 校验必填字段，通过后保存数据、关闭弹窗、刷新画布 |

# 个人方案模块

> 本模块为教师提供个人课程方案的管理能力，支持教师上传、编辑、删除自己准备的教学活动。个人方案中的活动可被引用到排课系统的活动列表中，供教师在排课时使用。

## 一、产品定位

「个人方案」是排课系统的子模块，面向一线教师，用于管理教师自主准备的教学活动。教师可以在此上传课程资料、填写活动信息，形成自己的活动资源库。个人方案中的活动默认来源为「我的」，可在排课模块的活动列表中直接拖拽使用。

## 四、筛选规则

### 4.1 活动名称检索

| 元素 | 说明 |
|------|------|

| 元素 | 说明 |

### 4.3 活动类型筛选

| 元素 | 说明 |
|------|------|
| 下拉选择器 | 选项：「全部」「生活」「运动」「游戏」「学习」 |

### 4.4 年级筛选

| 元素 | 说明 |
|------|------|

### 4.5 创建时间范围筛选

| 元素 | 说明 |
|------|------|
| 时间范围选择器 | 包含「开始日期」和「结束日期」两个日期输入框 |
| 占位文字 | 「开始日期」「结束日期」 |

### 4.6 筛选联动规则

所有筛选条件为**交集关系**，同时生效。无任何筛选条件时展示全部个人活动。

- 标签样式：圆角 4px，内边距 2px 8px，字号 12px
- 不同领域/类型/年级使用不同颜色区分

**操作列**
- 「详情」文字按钮：`#1890FF`，点击打开详情弹窗（只读）
- 「编辑」文字按钮：`#1890FF`，点击打开编辑弹窗
- 「删除」文字按钮：`#FF4D4F`，点击后二次确认
- 按钮间距 12px

### 空状态

- 无数据时：表格区域展示「暂无个人活动」居中提示
- 搜索/筛选无结果时：表格区域展示「未找到相关活动」居中提示

| 属性 | 说明 |
|------|------|
| 触发方式 | 点击「+ 上传资料」按钮，唤起系统文件选择器 |
| 支持格式 | 常见文档格式（PDF / Word / PPT / 图片 / 视频 / 音频等） |

| 删除操作 | 点击某项右侧 `✕` 按钮，移除该资料 |
| 文件大小限制 | 单个文件不超过 50MB（前端提示） |
| 数量限制 | 最多上传 10 个文件 |
